01. Enquadramento do Problema

Auditoria em Dados Não Estruturados

Este projeto propõe um modelo de Auditoria Algorítmica para mitigar a assimetria informacional em ecossistemas de Fintechs. Através do processamento de rastros digitais (repositórios de código), convertemos sinais técnicos brutos em indicadores de governança e saúde institucional.

PLN

Processamento de Linguagem Natural como motor de extração de evidências.

GovTech

Aplicação de transparência em ativos tecnológicos protegidos por sigilo.

02. Fluxo Metodológico (3 Etapas)
ETAPA 01

Extração e purificação com RE (Regex) e pré-processamento spaCy.

ETAPA 02

Aquisição de dados via Web Scraping (BeautifulSoup) e Web API (GitHub).

ETAPA 03

PLN Avançado: NER, POS, Lematização e análise via Transformers/Embeddings.

09. Glossário Técnico-Acadêmico de Metadados
PLN / NLP

Campo da IA focado na interpretação de textos humanos.

NER (Named Entity Recognition)

Identificação de nomes próprios, locais e tecnologias no corpus.

Lematização

Redução de palavras à sua base semântica (radical).

POS Tagging

Classificação gramatical (Verbos, Substantivos, Adjetivos).

Transformers

Arquiteturas neurais avançadas (ex: BERT) para análise contextual profunda.

Stopwords

Palavras sem valor semântico (artigos, preposições) filtradas na Etapa 1.

10. Considerações Finais e Entrega de Valor

O projeto validou a hipótese de que métricas extraídas de dados não estruturados são correlatas ao sucesso em rodadas de investimento e estabilidade de sistemas.

  • Mapeamento de Assimetria Informacional reduzido em 12.5%.
  • Modelo replicável para outros setores (LegalTech, HealthTech).
  • Framework 100% alinhado aos critérios de avaliação da banca 2026.
