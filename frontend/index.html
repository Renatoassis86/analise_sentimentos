Apresentação do Projeto

Arkos MI: Neural Marketing Intelligence para Startups

RA

Signal over Noise

O Arkos MI foi concebido para resolver a **Assimetria de Informação** no ecossistema de inovação. Em um cenário onde narrativas de marketing muitas vezes divergem da realidade técnica, nossa engine utiliza inteligência de sinais latentes para auditar a legitimidade de startups e empresas de tecnologia.

Desafio

Monitoramento de fontes massivas e não estruturadas que geram sobrecarga de decisão.

Solução

Automação ponta-a-ponta: da captura via API de alta frequência até a síntese prescrítiva.

Impacto

Visão antecipada de crises reputacionais e validação técnica para investidores (Due Diligence).

Roteiro de Apresentação Acadêmica

Mapeamento integral das **3 Etapas Regulamentares** do projeto e fundamentação teórica.

As 3 Etapas Exigidas

  • Etapa 01: NLP Base: Processamento de linguagem natural com Expressões Regulares (RE) e spaCy para purificação inicial.
  • Etapa 02: Coleta & Ingestão: Web scraping (BeautifulSoup) e coleta de dados via GitHub REST API v3 / Web APIs.
  • Etapa 03: NLP Avançado & IA: Tokenização, Normalização, Lematização, POS tagging, Noun Chunks, NER e Análise de Sentimento com **Embeddings & Transformers**.

Fórmulas & Teoria (Slide 1)

Health Index Algorithm:

Score = (S_stars * w1) + (S_commits * w2) - (S_issues * w3) + (S_sentiment * w4)

Parâmetros: w1=0.4, w2=0.3, w3=0.1, w4=0.2 (IA Bias).

Implementação (Slide 2)

A teoria foi implementada em módulos Python independentes em /analise_sentimentos/src/. Utilizamos pipeline("sentiment-analysis") do HuggingFace para abstração dos Transformers.

Dica de Estudo: Explore a documentação oficial em spacy.io e huggingface.co.

Diário de Bordo (Slide 3)

  • Erro: Vercel "Build Command Not Found" (Vite).
  • Solução: Bypass via vercel.json e package.json vazio para deploy estático.

Engenharia de Dados & ETL

A transformacão do dado bruto em inteligência acionável segue o processo rigoroso de **Extração, Transformação e Carga (ETL)**.

1. Extract

Acessamos o objeto JSON da API do GitHub. Campos capturados:

  • stargazers_count: Proxy de validação de rede.
  • forks_count: Proxy de reprodutibilidade técnica.
  • open_issues: Métrica de débito operacional.

2. Transform

Onde ocorre o cálculo do **Health Index**. Algoritmo Arkos:

  • Normalização: Escalonamento de 0 a 100.
  • Sentiment Bias: O score final é ponderado pelo resultado da análise de texto (BERT).
  • NER Injection: Identificação de conceitos-chave.

3. Load

Sincronização com o repositório de dados do Dashboard:

  • Single Source of Truth: Geração do data.json.
  • Atomic Update: Escrita atômica para evitar corrupção durante o deploy.

Resiliência de Ingestão

O Arkos MI não depende de uma única fonte. Nossa estratégia de coleta utiliza:

  • GitHub REST V3: Autenticação via Token pessoal para evitar Rate Limits (60 vs 5000 req/hr).
  • BrasilAPI: Consulta direta a dados de CNPJ para validar a existência legal da entidade.
  • Headers Polymorphism: Uso de User-Agents dinâmicos para scraping de fóruns e sites especializados, simulando navegação humana.

Segurança de Dados

Nenhum segredo (tokens/senhas) é exposto no Frontend. Todo o processamento de chaves ocorre no backend seguro (Python) em máquinas protegidas.

Deep Feature Extraction

Utilizamos a biblioteca **spaCy** com o modelo `pt_core_news_lg` para processamento linguístico de alta precisão em português do Brasil.

Mapa de Entidades (NER)

Etapas do Pipeline NLP

  • Tokenização & Lemmatização: Redução de palavras ao seu radical para análise semântica.
  • Noun Chunks: Extração de frases nominais inteiras para captar nomes de produtos e concorrentes.
  • POS Tagging: Identificamos se o mercado fala da startup usando "verbos de ação" (inovação) ou "adjetivos vagos" (hype).

Análise Preditiva e Redes Neurais

A análise de sentimento tradicional (baseada em listas de palavras) é falha em contextos complexos. Resolvemos isso com **Transformers**.

Estudo de Ablação (Baseline vs AI)

Por que BERT?

Diferente do spaCy vanilla, o **BERT (Bidirectional Encoder Representations from Transformers)** entende o contexto global da frase.

Exemplo: "A taxa de juros subiu e isso é terrível." vs "A confiança na startup subiu e isso é excelente."
O BERT entende que "Subiu" tem polaridades diferentes dependendo do substantivo acompanhante.
